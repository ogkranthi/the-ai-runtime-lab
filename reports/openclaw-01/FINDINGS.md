## Working notes

Hand-written analysis lives here. `run-all.sh` regenerates the table above and
leaves everything from this heading down untouched.

### 102323, HEIC and TIFF media type

The adapter forwards the original media_type to Anthropic. Anthropic supports a
fixed set of image media types; HEIC and TIFF are not in it, so the request comes
back 400 and the user gets nothing useful. The same fixtures succeed on OpenAI and
Gemini, which is the control that shows this is adapter behavior, not a bad file.

### 102324, tool_result image to a text-only model

When a tool returns an image block and the session targets a text-only model, the
adapter still sends the image block and the provider rejects it. The fix is a
capability check before the send, not after the 400.

### 102321, OpenAI refusal dropped

The chat-completions response carries a refusal field, but the adapter maps only
the assistant content to the user. A refusal has empty content, so the user sees
an empty reply with no reason. The refusal text never reaches the channel.

### 102320, Gemini provider-prefixed model id

A provider-prefixed model id passes normal chat but fails the multimodal
capability check on the functionResponse path, because the check compares against
the unprefixed id. The unprefixed control passes the same path.
